<div class="para" label-module="para">作为开源代码库以及版本控制系统，Github拥有超过900万开发者用户。随着越来越多的应用程序转移到了云上，Github已经成为了管理软件开发以及发现已有代码的首选方法。</div>
<div class="para" label-module="para">如前所述，作为一个分布式的版本控制系统，在Git中并不存在主库这样的概念，每一份<a target=_blank href="/item/%E5%A4%8D%E5%88%B6">复制</a>出的库都可以独立使用，任何两个库之间的不一致之处都可以进行合并。</div>
<div class="para" label-module="para">GitHub可以托管各种git库，并提供一个web界面，但它与外国的<a target=_blank href="/item/SourceForge/6562141" data-lemmaid="6562141">SourceForge</a>、<a target=_blank href="/item/Google%20Code">Google Code</a>或中国的<a target=_blank href="/item/coding/8921246" data-lemmaid="8921246">coding</a>的服务不同，GitHub的独特卖点在于从另外一个项目进行分支的简易性。为一个项目贡献代码非常简单：首先点击项目站点的“fork”的按钮，然后将代码检出并将修改加入到刚才分出的代码库中，最后通过内建的“pull request”机制向项目负责人申请代码合并。已经有人将GitHub称为代码玩家的MySpace。</div>
<div class="para" label-module="para">在GitHub进行分支就像在<a target=_blank href="/item/Myspace">Myspace</a>（或<a target=_blank href="/item/Facebook">Facebook</a>…）进行交友一样，在社会关系图的节点中不断的连线。</div>
<div class="para" label-module="para">GitHub项目本身自然而然的也在GitHub上进行托管，只不过在一个私<div class="lemma-picture text-pic layout-right" style="width:250px; float: right;">
<a class="image-link" nslog-type="9317" 
			href="/pic/github/10145341/0/5366d0160924ab181d75807d34fae6cd7b890b1c?fr=lemma&ct=single" target="_blank"
		title="" style="width:250px;height:100px;">
<img  class="lazy-img" src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAMAAAAoyzS7AAAAGXRFWHRTb2Z0d2FyZQBBZG9iZSBJbWFnZVJlYWR5ccllPAAAAAZQTFRF9fX1AAAA0VQI3QAAAAxJREFUeNpiYAAIMAAAAgABT21Z4QAAAABJRU5ErkJggg==" data-src="https://gss2.bdstatic.com/9fo3dSag_xI4khGkpoWK1HF6hhy/baike/s%3D250/sign=3660e54f32fa828bd5239ae6cd1e41cd/5366d0160924ab181d75807d34fae6cd7b890b1c.jpg"  alt="" style="width:250px;height:100px;"/>
</a>
</div>有的，公共视图不可见的库中。<a target=_blank href="/item/%E5%BC%80%E6%BA%90%E9%A1%B9%E7%9B%AE">开源项目</a>可以免费托管，但私有库则并不如此。Chris Wanstrath，GitHub的开发者之一，肯定了通过付费的私有库来在财务上支持免费库的托管这一计划。</div>
<div class="para" label-module="para">是的，我们正是这么计划的。通过与客户的接洽，开发FamSpam，甚至是开发GitHub本身，GitHub的私有库已经被证明了物有所值。任何希望节省时间并希望和团队其它成员一样远离页面频繁转换之苦的人士都会从GitHub中获得他们真正想要的价值。</div>
<div class="para" label-module="para">在GitHub，用户可以十分轻易地找到海量的<a target=_blank href="/item/%E5%BC%80%E6%BA%90/20720669" data-lemmaid="20720669">开源</a>代码。</div><div class="anchor-list">
<a name="2" class="lemma-anchor para-title" ></a>
<a name="sub3366456_2" class="lemma-anchor " ></a>
<a name="版本历史" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">github</span>版本历史</h2>
<a class="edit-icon j-edit-link" data-edit-dl="2" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
</div>
<div class="para" label-module="para">2012年12月24日，GitLab 4.0 RC2 发布，<a target=_blank href="/item/%E5%BC%80%E6%BA%90/20720669" data-lemmaid="20720669">开源</a>的 Github<a target=_blank href="/item/%E5%85%8B%E9%9A%86">克隆</a>。</div><div class="anchor-list">
<a name="3" class="lemma-anchor para-title" ></a>
<a name="sub3366456_3" class="lemma-anchor " ></a>
<a name="内幕信息" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">github</span>内幕信息</h2>
<a class="edit-icon j-edit-link" data-edit-dl="3" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
</div>
<div class="para" label-module="para">Chris Wanstrath还向记者分享了关于GitHub的一些内幕信息︰</div>
<div class="para" label-module="para">GitHub主要用Rails实现。我们在进行的post-commit集成小应用完全使用Merb编写。我们使用了<a target=_blank href="/item/Python/407313" data-lemmaid="407313">Python</a>的Pygments来做格式高亮显示，另外，还用了Ara T. Howard&#39;s Bj加上一些Ruby<a target=_blank href="/item/%E8%84%9A%E6%9C%AC">脚本</a>来做我们的<a target=_blank href="/item/%E6%8E%92%E9%98%9F%E7%B3%BB%E7%BB%9F">排队系统</a>。当然，我们用了Ruby Grit库来和Git进行交互。</div>
<div class="para" label-module="para">GitHub已经有了一组引人注目的特性，除了命令式的库<a target=_blank href="/item/%E6%B5%8F%E8%A7%88%E5%99%A8">浏览器</a>和一个项目Wiki，GitHub甚至还包括了一个GitHub gem，以使通过shell方式使用GitHub更为方便。更多的未来特性已经在计划中︰</div>
<div class="para" label-module="para">许多人都希望能有一个条目系统，因此一个简单的条目系统已经在开发中。此外，正如我前面所言，我们尚在进行<a target=_blank href="/item/RubyGems">RubyGems</a>服务器和一些之前留出的post-commit钩子方面的<a target=_blank href="/item/%E5%B7%A5%E4%BD%9C">工作</a>。如果你不能或就是不想托管一个你自己的<a target=_blank href="/item/%E5%AE%88%E6%8A%A4%E8%BF%9B%E7%A8%8B">守护进程</a>，你可以使用我们所提供的。</div>
<div class="para" label-module="para">我们还在开发一些特性来帮助公司在使用Github时可以停留在sync之上。</div>
<div class="para" label-module="para">最后，我们也在进行<a target=_blank href="/item/API">API</a>发布方面的<a target=_blank href="/item/%E5%B7%A5%E4%BD%9C">工作</a>。我们很快就会发布一些只读性的API，随后是一些很强大的“写”集成。你可以使用API将新的事件发布到新闻feed中，发消息和做其他许多很酷的事情。</div>
<div class="para" label-module="para">GitHub尚未设定官方版本的发布日期，不过估计在三月底（GitHub已经上线，但只能通过邀请注册）。更多关于GitHub的信息可以参见GitHub官方网站或GitHub博客。通过GitHub进行代码管理的开源项目列表也已经可以查阅。</div><div class="anchor-list">
<a name="4" class="lemma-anchor para-title" ></a>
<a name="sub3366456_4" class="lemma-anchor " ></a>
<a name="Windows应用" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">github</span>Windows应用</h2>
<a class="edit-icon j-edit-link" data-edit-dl="4" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
</div>
<div class="para" label-module="para">GitHub 使用 git 分布式版本控制系统，而 git 最初是 LinusTorvalds 为帮助<a target=_blank href="/item/Linux">Linux</a>开发而创造的，它针对的是 Linux 平台，因此 git 和 Windows 从来不是最好的朋友，因为它一点也不像 <a target=_blank href="/item/Windows">Windows</a>。GitHub 发布了GitHub for Windows，为 Windows 平台开发者提供了一个易于使用的 Git 图形客户端。<div class="lemma-picture text-pic layout-right" style="width:220px; float: right;">
<a class="image-link" nslog-type="9317"

<div class="para" label-module="para">GitHub for <a target=_blank href="/item/Windows">Windows</a> 是一个 Metro 风格<a target=_blank href="/item/%E5%BA%94%E7%94%A8%E7%A8%8B%E5%BA%8F">应用程序</a>，<a target=_blank href="/item/%E9%9B%86%E6%88%90">集成</a>了自包含版本的 Git，bash 命令行 shell，PowerShell 的 posh-git 扩展。GitHub 为 Windows 用户提供了一个基本的图形前端去处理大部分常用版本控制任务，可以创建版本库，向本地版本库递交补丁，在<a target=_blank href="/item/%E6%9C%AC%E5%9C%B0">本地</a>和<a target=_blank href="/item/%E8%BF%9C%E7%A8%8B">远程</a>版本库之间同步。<a target=_blank href="/item/%E5%BE%AE%E8%BD%AF">微软</a>也通过<a target=_blank href="/item/CodePlex">CodePlex</a>向开发者提供 git 版本控制系统，而 GitHub 创造了一个更具有吸引力的 Windows 版本。</div><div class="anchor-list">

<a name="配置管理" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">github</span>配置管理</h2>
<a class="edit-icon j-edit-link" data-edit-dl="5" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
</div>
<div class="para" label-module="para">GitHub上已自动配置的<a target=_blank href="/item/Mac/173" data-lemmaid="173">Mac</a><a target=_blank href="/item/%E7%AC%94%E8%AE%B0%E6%9C%AC%E7%94%B5%E8%84%91">笔记本电脑</a>，一个工具，可以转换设置<a target=_blank href="/item/Linux/27050" data-lemmaid="27050">Linux</a>或Windows机器。<br/>　　BOXEN是GitHub的自动化工具，设置和配置的Mac笔记本电脑<a target=_blank href="/item/%E8%BD%AF%E4%BB%B6%E5%BC%80%E5%8F%91">软件开发</a>或其他类型的工作，正在使用他们的开发人员，律师，设计师，付货人，等。我们的想法是准备系统以自动方式和作为无差错尽可能用最少的干预工作。根据GitHub上，与一个新的开发机器上，他的Mac系统成立，并准备在30分钟内提交代码。<br/>　　BOXEN的基础上收集了大量的几十个木偶模块，使设置的各种软件，如卡桑德拉，MongoDB中，Java软件中，<a target=_blank href="/item/Python/407313" data-lemmaid="407313">Python</a>和<a target=_blank href="/item/Ruby/11419" data-lemmaid="11419">Ruby</a>开发中，节点，JS，nginx的，<a target=_blank href="/item/Skype">Skype</a>公司，甚至<a target=_blank href="/item/MINECRAFT">MINECRAFT</a>。虽然机器上配备了一个预配置，每个用户都可以调整它的配置应有的作用。</div><div class="anchor-list">
<a name="6" class="lemma-anchor para-title" ></a>

<a name="融资" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">github</span>融资</h2>
<a class="edit-icon j-edit-link" data-edit-dl="6" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
</div>
<div class="para" label-module="para">全球最大的社交<a target=_blank href="/item/%E7%BC%96%E7%A8%8B">编程</a>及代码托管网站GitHub以其开创性的新型软件开发方式并且能高效利用有限的资源通过自力更生实现公司盈利和300%的年收入增长成功的吸引知名风投机构Andreessen Horowitz一亿美金的投资。新的资金注入将帮助GitHub平台得到进一步的改进和扩展。</div>
<div class="para" label-module="para">事实上，这不仅对首次接受外部投资的GitHub意义重大，同样对于投资方Andreessen Horowitz而言，这也是其迄今为止进行过的最大一次单笔投资案。作为投资案的一部分，Andreessen Horowitz的<a target=_blank href="/item/%E5%90%88%E4%BC%99%E4%BA%BA">合伙人</a>Peter Levine将入主GitHub董事会。</div>
<div class="para" label-module="para">根据GitHub官方解释，这笔资金除了用于扩充员工队伍，改进现有服务并移植到<a target=_blank href="/item/%E7%A7%BB">移</a><div class="lemma-picture text-pic layout-right" style="width:220px; float: right;">

<div class="para" label-module="para">GitHub有170万名软件开发人员的忠实用户，他们平均每天更新8万个并新建7千个<a target=_blank href="/item/%E8%BD%AF%E4%BB%B6%E5%BA%93">软件库</a>。对GitHub网站上托管的总计超过300万个软件库，其联合创始人Chris Wanstrath曾经形象地称其为“<a target=_blank href="/item/%E7%A8%8B%E5%BA%8F%E5%91%98">程序员</a>的<a target=_blank href="/item/%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91">维基百科</a>全书”。</div><div class="anchor-list">

<a name="有关报道" class="lemma-anchor " ></a>
</div><div class="para-title level-2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">github</span>有关报道</h2>
<a class="edit-icon j-edit-link" data-edit-dl="7" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
</div>
<div class="para" label-module="para">2013年1月15日晚间，全球最大的社交编程及代码托管网站GitHub突然疑似遭遇<a target=_blank href="/item/DDOS%E6%94%BB%E5%87%BB">DDOS攻击</a>，访问大幅放缓，该网站管理员经过日志查询，发现是来自12306的抢票插件用户洪水般的访问导致GitHub出现问题。</div>
<div class="para" label-module="para">2013年1月20日左右，GitHub在<a target=_blank href="/item/%E4%B8%AD%E5%9B%BD%E5%A4%A7%E9%99%86">中国大陆</a>被官方的防火长城封锁（封锁手段为域名污染和关键词过滤），李开复等一些微博大号呼吁解禁GitHub，微博转发近10万条。2013年1月23日github网站恢复正常访问。</div>
<div class="para" label-module="para">2014年1月23日联合创始人汤姆·普雷斯顿-维尔纳（Tom Preston-Werner）从另一位联合创始人克里斯·万斯特拉斯（Chris Wanstrath）手中接过总裁职位，后者也将接过普雷斯顿-维尔纳留下的CEO位置。</div>
<div class="para" label-module="para">2018年6月4日晚，微软宣布，通过75亿美元的股票交易收购代码托管平台GitHub。<sup class="sup--normal" data-sup="3" data-ctrmap=":3,">
[3]</sup><a class="sup-anchor" name="ref_[3]_3366456">&nbsp;</a>
</div>
<div class="para" label-module="para">2018年10月26日，微软以75亿美元收购GitHub交易已完成。同年10月29日微软开发者服务副总裁奈特·弗里德曼(Nat Friedman)将成为GitHub的新一任CEO。<sup class="sup--normal" data-sup="4" data-ctrmap=":4,">
